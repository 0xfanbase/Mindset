- **2026-07-22, v1.25 — Fable's round-one pre-merge audit: ship after two required fixes, both found by measuring the live render, not by reading the source diff.**
  1. Every detail photo was rendering as a 350×600 portrait crop instead of the intended 3:2 landscape frame. Root cause, confirmed by measurement rather than assumed: the `<img>` element's `height="600"` HTML attribute (kept deliberately, feeding the pre-load size hint alongside `width="900"`) maps to a browser presentational hint for the `height` CSS property; since `.mara-photo-img`'s own rule never declared `height` at all, that hint stood completely unopposed, and CSS `aspect-ratio` only auto-computes a dimension that is otherwise unset — with `height` pinned at 600px by the hint, `aspect-ratio: 3/2` against a ~350px-wide mobile column produced a 350×600 box instead of 350×233. - **2026-07-22, v1.25 — a Playwright testing lesson, logged since it cost real debugging time and will recur:** the first two attempts at writing a regression test for the scroll-restore fix both produced false failures that looked like real bugs (a restored scroll position wildly different from the value manually set beforehand). Root cause: `page.click(selector)` performs its own actionability auto-scroll, bringing the target element into view (and, evidence suggests, roughly toward viewport center) *before* the click event fires — when the manually-set scroll position didn't happen to make the target fully visible (most dramatically when the target was the 20th/last tile in the grid), Playwright's own auto-scroll moved the page further than the test's setup intended, and the test then measured the *real* app behavior against the *wrong* expected value. Confirmed by adding temporary trace logging directly inside `selectAnimal()`: `window.scrollY` at function entry already reflected Playwright's auto-scroll, before any of the tab's own code had run. Resolved by testing with raw coordinate taps (`page.mouse.click(x, y)` computed from the target's own `getBoundingClientRect()`, and separately corroborated via DOM `element.click()`) instead of `page.click(selector)` — both bypass Playwright's smart-scroll and simply act wherever the element visually sits, matching what a real finger-tap does. With that correction, the restore is pixel-exact (verified at multiple scroll depths, not just one). Fable's second-round audit independently re-derived and confirmed the same finding using its own DOM-`element.click()` method, arriving at it separately rather than simply trusting this session's account.
